4.1

Lightweight, Multi-Factor Authentication and Attribute-Based Authorization for Secure Substation Automation in Smart Grid Networks

Supported by: Ontario Ministry of Research and Innovation and NSERC Discovery Grant

Executive Overview

This comprehensive project addresses critical cybersecurity gaps in modern Substation Automation Systems (SAS). As smart grids increasingly adopt IP-based communication and open protocols like IEC 61850 and DNP3, they become vulnerable to attacks due to reliance on weak authentication mechanisms—such as static or shared passwords—and a lack of fine-grained, auditable access control. These vulnerabilities conflict with stringent regulatory requirements from NERC CIP and IEC 62351, which mandate resilient, identity-based security even during network outages.

Core Contribution

The core contribution of this work is a lightweight, substation-level security architecture that integrates multi-factor authentication, attribute-based authorization, and offline-capable operation while maintaining compatibility with existing infrastructure. The system operates without requiring costly hardware replacements, making it suitable for incremental deployment in operational technology (OT) environments.

Key Innovations

Dual-Mode Authentication

Supports both normal operation and degraded mode during WAN outages, ensuring continuous secure access.

Elliptic Curve Cryptography

Uses secp256r1 with implicit certificates, reducing certificate size to ~400 bytes and eliminating explicit signature overhead.

Zero-Knowledge Proof Protocol

Incorporates ZKP-inspired protocol based on Schnorr identification scheme for enhanced security.

Attribute-Based Authorization

Implements Attribute Certificates (ACs) for fine-grained access control with short-lived validity periods.

System Architecture

The architecture centers on a Substation Controller (SSC) that acts as a local security gateway, performing server-aided verification (SAV). It supports two complementary authentication schemes:

A

Scheme A: Normal Operation

The user authenticates through the SSC, which mediates access to Intelligent Electronic Devices (IEDs).

B

Scheme B: Degraded Operation

During WAN outages, the user connects directly to an IED, which collaborates with the SSC using cached credentials to complete verification autonomously.

Technical Implementation and Simulation

To validate feasibility, the team built a detailed simulation framework in NS-3 (v3.39+), incorporating realistic constraints and modern cryptographic libraries. The NS-3 model included three node types:

  • UserNode: Simulated remote operator with multi-factor credentials.
  • SscNode: Full-featured controller acting as DNP3 master and local attribute authority.
  • IedNode: Resource-constrained (500 MHz CPU, 64 MB RAM) DNP3 outstation representing real-world IEDs.

Performance Results

Authentication Latency

~120 ms (Scheme A)
~150 ms (Scheme B)

IED CPU Utilization

<15% (Scheme A)
<18% (Scheme B)

Message Overhead

Only +128 to +144 bytes per transaction

Success Rate

100% in normal conditions
95%+ during link failures

Security and Compliance

Impersonation Resistance

Requires possession of ECC private key plus two passwords.

MitM & Replay Protection

Prevented through challenge-response protocols and session-unique nonces.

Insider Abuse Mitigation

Addressed via individualized ACs and cryptographic non-repudiation.

DoS Protection

Limited by rate-limiting and pre-validation at the SSC.

Conclusion

This project delivers a practical, standards-compliant, and resilient security solution for substation automation. By combining lightweight ECC, multi-factor authentication, attribute-based authorization, and offline resilience, it enables utilities to meet stringent regulatory mandates without costly infrastructure overhauls. The work not only advances the state of the art in ICS security but also demonstrates how rigorous cryptographic design can be adapted for real-world operational constraints—offering a blueprint for securing the evolving smart grid and beyond.

4.2

Secure, Scalable, and Interoperable Public Key Infrastructure for Multi-Domain Vehicle-to-Grid (V2G) Networks

Supported by: Ontario Ministry of Economic Development, Innovation and Trade and Ontario ENERGY Smart Grid Fund

Executive Overview

This project addresses critical security and scalability challenges in Vehicle-to-Grid (V2G) ecosystems. As electric vehicles (EVs) increasingly participate in smart grid operations—providing energy storage, frequency regulation, and bidirectional power flow—secure, dynamic, and cross-jurisdictional authentication becomes essential. However, the current V2G security standard, ISO/IEC 15118, relies on conventional X.509-based Public Key Infrastructure (PKI), which suffers from significant limitations: large certificate sizes (up to 800 bytes), rigid hierarchical trust models, slow validation due to explicit signatures, and path-length restrictions (≤3 levels). These constraints hinder seamless roaming of EVs across administrative domains such as different utilities, provinces, or countries.

Solution Overview

To overcome these challenges, the project proposes a hybrid, multi-domain PKI framework grounded in Elliptic Curve Cryptography (ECC) and implicit certificates, specifically leveraging the ECQV (Elliptic Curve Qu-Vanstone) self-certified model. In this approach, a user's public key is reconstructed directly from the certificate itself, eliminating the need to transmit or store public keys separately and removing explicit signature overhead.

Key Innovations

Compact Certificates

Certificates ~50% smaller than ECDSA and ~85–90% smaller than RSA-1024, with sizes well below 400 bytes.

Dual-Layer Trust Model

Combines hierarchical PKI within domains with peer-to-peer cross-certification between domains.

Cross-Domain Validation

Novel algorithms for efficient path validation across domains with zero-knowledge-inspired techniques.

Enhanced Scalability

EVs can store more than the ISO/IEC 15118–mandated limit of five root certificates.

Performance Results

Certificate Validation Time

60–70% reduction compared to traditional X.509 RSA-1024

Certificate Size

<400 bytes (vs. 800+ bytes for traditional certificates)

Cross-Domain Efficiency

Performance gains increase as certificate chains lengthen

Publications

This work was published across three key papers:

  • GLOBECOM 2011: Initial framework for multi-domain V2G PKI
  • DIVANet 2014: Enhanced algorithms for cross-domain validation
  • MILCOM 2015: Application to military and critical infrastructure scenarios

Impact and Applicability

Beyond V2G, the framework is applicable to other cyber-physical systems—including smart home energy networks and industrial IoT—where lightweight, cross-domain PKI is needed. By replacing rigid X.509 with a flexible, implicit-certificate-based model, the project enables the secure, dynamic, and efficient trust management essential for the future smart grid.

4.3

Secure and Lightweight Authentication and Authorization Framework for Smart Energy Home Area Networks (SE-HAN)

Supported by: Ontario Ministry of Research and Innovation and NSERC Discovery Grant

Executive Overview

This project addresses critical security challenges in Smart Energy Home Area Networks (SE-HANs). SE-HANs form a vital link in the Advanced Metering Infrastructure (AMI) of the smart grid, connecting utility-owned devices (e.g., smart meters, Energy Service Interfaces—ESIs) with consumer-owned appliances (e.g., thermostats, EV chargers). However, their heterogeneous, resource-constrained nature makes them vulnerable to man-in-the-middle (MitM), impersonation, session hijacking, and key compromise attacks—especially when relying on traditional pre-shared key (PSK) models, which are inflexible and unscalable in multi-vendor environments.

Technical Approach

To overcome these limitations, the project introduces a comprehensive, lightweight, and ECC-based security architecture grounded in Elliptic Curve Cryptography (ECC) and self-certified implicit certificates (based on the ECQV—Elliptic Curve Qu-Vanstone—scheme). This approach eliminates explicit CA signatures, reduces certificate size by ~50% compared to ECDSA, and requires only three ECC point multiplications per authentication—making it highly efficient for constrained devices.

Core Mechanisms

Lightweight Device Authentication

Mutually authenticated key agreement protocol across five phases with strong security properties including UKS-R, FS, KCI-R, and KKS.

Secure Remote Access Architecture

Three-party model where ESI acts as gatekeeper between external users and internal HAN devices using modified GPS zero-knowledge identification protocol.

ECC-Based Digital Signature Scheme

Built from identification protocol via Fiat–Shamir heuristic, providing confidentiality, unforgeability, and non-repudiation for remote commands.

Key Advantages

  • No long-term pre-shared keys: Enables scalable, multi-vendor deployment
  • Compact credentials: Minimal communication rounds, reducing bandwidth and energy use
  • Remote access support: Ubiquitous access from smartphones and third-party service providers
  • Standards alignment: Compatible with ZigBee Smart Energy Profile (SEP) standards

Publications

  • IEEE ICCE 2011: Lightweight Device Authentication protocol
  • IEEE ISGT 2012: Secure Remote Access Architecture
  • Springer Journal of Supercomputing 2013: ECC-Based Digital Signature Scheme

Broader Impact

Beyond residential energy management, this framework laid the cryptographic groundwork for later WiSense projects on V2G security and substation automation, demonstrating how lightweight, zero-trust principles can be applied across the smart grid ecosystem—from home networks to critical infrastructure.

4.4

Lightweight, Secure, and Privacy-Preserving Communication Architectures for Wireless Mesh AMI and Aggregator-Based V2G Networks

Supported by: Ontario Ministry of Research and Innovation and NSERC Discovery Grant

Executive Overview

This project addresses two critical security and privacy challenges in smart grid communications: (1) securing wireless mesh Advanced Metering Infrastructure (AMI) networks against active routing attacks, and (2) enabling privacy-preserving participation of electric vehicles (EVs) in Aggregator-Based Vehicle-to-Grid (V2G) systems.

Wireless Mesh AMI Security

For Wireless Mesh AMI, traditional routing protocols like AODV are vulnerable to black hole, wormhole, and Sybil attacks, especially in dynamic, multi-hop environments where mobile maintenance vehicles act as transient repeaters. To counter this, the team designed two novel protocols: SAODV2MP and its interference-aware variant iSAODV2MP.

Protocol Features

Aggregated Designated Verifier Signatures (A-DVS)

ECC-based signatures for Route Request (RREQ) integrity, verifiable only by the destination and resistant to tampering.

Aggregated MACs with On-the-Fly Shared Keys (AMAC-OSK)

Pairwise ephemeral keys (via ECDH) enable constant-size, aggregated message authentication for Route Replies (RREPs), regardless of path length.

Watchdog Monitoring

In iSAODV2MP detects malicious relay behavior like packet dropping.

Performance Results

Packet Delivery Ratio

>80% even under 20% malicious nodes

Computational Overhead

Only 3–4 ECC point multiplications per node

Performance Advantage

Significantly outperforms SAODV and SAODV-MAP in dynamic, attack-prone environments

Privacy-Preserving V2G

For Aggregator-Based V2G, the challenge lies in balancing authentication with EV owner privacy. Existing certificate- or identity-based schemes either leak user identity or rely on computationally expensive bilinear pairings, unsuitable for embedded EV hardware. The project introduced an ECC-based Restrictive Partially Blind Signature (RPBS) scheme that enables anonymous yet accountable grid participation.

RPBS Scheme Features

Dynamic Pseudonyms

EVs use unlinkable pseudonyms (PIDEV) across sessions.

Self-Certified ECC PKI

TA issues partial private keys without key escrow.

Conditional Identity Revelation

Supports identity revelation in cases of misuse while maintaining privacy otherwise.

Computational Efficiency

Avoids bilinear pairings, using only ECC scalar multiplications.

Performance Advantage

The RPBS scheme uses only ECC scalar multiplications (~0.6 ms per operation vs. ~4.5 ms for pairing-based approaches), resulting in >50% computational savings over prior ID-based schemes.

Unified Design Philosophy

Both components reflect a unified design philosophy: minimize reliance on heavy PKI or trusted third parties, leverage lightweight ECC, and support offline or degraded-mode operation—essential for real-world smart grid resilience. The AMI solutions align with IEEE 802.11s, ZigBee, and AODVv2, while the V2G architecture extends ISO/IEC 15118 with robust privacy.

4.5

Lightweight, Mutual Two-Factor Authentication with Session Key Agreement for Resource-Constrained Wireless Sensor Networks

Funded by: Ontario Ministry of Research and Innovation

Executive Overview

This research project addresses critical security gaps in Wireless Sensor Networks (WSNs) deployed in unattended or infrastructure-limited environments—such as disaster zones, battlefields, or remote industrial sites. In such settings, users must securely authenticate directly with sensor nodes, often without a persistent gateway or base station, yet existing two-factor authentication schemes suffer from severe vulnerabilities: susceptibility to stolen smart card attacks, lack of mutual authentication, absence of session key agreement, and insecure password update mechanisms.

Protocol Design

To overcome these limitations, the project proposes a novel, lightweight, mutual two-factor authentication protocol that integrates secure session key establishment while remaining feasible for highly constrained sensor hardware. The design uses only hash functions and XOR operations—avoiding public-key cryptography on sensor nodes—and supports offline password updates, enhancing both security and usability.

Protocol Phases

1

Registration

User registers with Gateway (GWN), which issues a smart card with carefully constructed parameters. The shared secret xs is never stored on the card.

2

Login & Authentication

User inputs identity and password; smart card verifies credentials and generates dynamic identity and authentication tokens with fresh nonces and timestamps.

3

Three-Party Mutual Authentication

User, GWN, and Sensor engage in synchronized exchange with chained hash-based challenges for mutual trust verification.

4

Session Key Agreement

User and sensor independently compute shared session key SK using pseudo-random function derived from dynamic values and secret xs.

5

Password Change

Users update passwords locally without GWN involvement, reducing network dependency and mitigating insider threats.

Security Properties

The protocol satisfies 10 essential security properties, including resistance to:

Stolen Smart Card Attacks

Shared secret xs never stored on card

Node Capture Attacks

Forward secrecy maintained

Replay & Parallel Session Attacks

Fresh nonces and timestamps prevent replay

Privileged Insider Attacks

Zero-knowledge password verification

Performance Analysis

Sensor Node Operations

Only 3 hash operations per session

User Device Operations

~9 hash operations (manageable on smartphones/PDAs)

Hardware Compatibility

Well within capabilities of IEEE 802.15.4/ZigBee-class devices

Formal Validation

Verified using Rubin's Non-Monotonic Cryptographic Protocol Logic

Application and Impact

Compared to prior work, this solution offers superior security with minimal overhead, striking an optimal balance for real-world WSN deployments. Its gateway-resilient design makes it particularly suitable for degraded or disconnected operational scenarios, aligning with the needs of smart grid edge networks, industrial IoT, emergency response, and environmental monitoring.

By closing critical security gaps while respecting hardware constraints, this work provides a practical, provably secure foundation for authenticated, encrypted interactions in next-generation wireless sensor ecosystems.
